NodeJS is an open source cross platform javascript run time built on Chrome v8 engine which
enables developers to run javascript outside the browser to build fast scalable server side 
applications.

Rest Apis Allow communication between client and server over Http allowing exchange of data in
json format using standard web protocols


Http is a core internet protocol that define how data is exchanged between client and 
servers on the web page
Common Http Method
Get Retrieves resources with 200 ok or 404 not found
Post creates a new resources and on success returns 201
Put is used to update or create a resource sending the complete resource replacing the 
specified one on the URL
Patch is used for partial updates sending only the part needing updates
Delete is used to delete a resource identified by a URI on success returns 200

package.json is a central configuration file that defines how the application is run, built 
and managed in NodeJS.
It stores esential information & configuration settings, manages dev tools and dependencies
defines executable script for tasks like run, buld, specifies entry point & run time behaviour

Metadata is data that provides information of other datas it provides context to raw data
providing meaning helps in seo and accurate interpretation of data.
Types
Preservation ensures long time access to digital assets by storing format details
Preservation Method & migration strategies.
Descriptive describe content details like author title keywords etc used to identify resources
Technical covers technical details like file size resolution software used in creating it
Structural covers the organization of data helping naviagation of pages or chapters 
Administrative covers management details like ownership creation date access rights etc

Package is a reusable piece of code that add functionality to a application
